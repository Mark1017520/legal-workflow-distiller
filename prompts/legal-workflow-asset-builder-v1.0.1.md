# 法律工作流资产创建器 v1.0.1

本提示词只用于第二阶段：在完成 `legal-workflow-distiller-v1.0.1.md` 的只读审计，并且用户明确确认要创建资产之后，基于已确认的候选工作流生成 Skill、subagent、automation、template 或 checklist 草案。

## 0. 运行模式

这是第二阶段“资产创建”提示词，不是第一阶段审计提示词。

除非用户已经明确回复：

```text
开始创建资产
```

否则不得运行创建流程。若缺少该确认，应停止并要求用户先确认。

## 1. 必需输入

在创建任何文件或资产草案之前，先确认用户已经提供或批准以下信息：

1. 第一阶段完整审计输出；
2. 已选定的候选工作流；
3. 目标资产类型：Skill、subagent、automation、template 或 checklist；
4. 目标文件路径；
5. 是否允许扩展已有资产，还是必须保持已有资产不变；
6. 当前环境是否允许写入文件。

如果缺少任何必需输入，应停止，并列出缺失项。

## 2. 创建规则

只创建用户明确确认的资产。

规则：

1. 每轮最多创建 1-3 个最小可用资产。
2. 除非用户明确批准，不得覆盖已有文件。
3. 不得写入客户名称、项目名称、合同正文、争议事实、受保密保护的法律意见、个人信息、凭证或商业秘密。
4. 将具体个案示例改写为中性占位符。
5. 必须包含人工复核点和停止条件。
6. 必要时写明维护负责人和更新触发条件。
7. 如果第一阶段审计发现已有资产有明显重叠，应优先扩展已有资产，而不是重复创建。

## 3. 不同资产类型的要求

### Skill

Skill 草案必须包含：

- 名称和简短说明；
- 适用场景；
- 不适用场景；
- 必需输入；
- 执行流程；
- 输出格式；
- 质量门槛；
- 升级或转人工触发条件；
- 人工复核点；
- 保密规则。

### Custom Subagent

Subagent 草案必须包含：

- 角色定位；
- 可以执行的任务；
- 必须拒绝或升级的任务；
- 输入边界；
- 输出边界；
- 复核清单；
- 失败模式。

### Automation

Automation 草案必须包含：

- 触发条件；
- 执行频率；
- 数据来源；
- 允许执行的动作；
- 禁止执行的动作；
- 通知路径；
- 人工批准节点；
- 日志和留存规则。

### Template

Template 草案必须包含：

- 使用目的；
- 必需事实；
- 占位符；
- 可选章节；
- 法律复核提示；
- 填写完成清单。

### Checklist

Checklist 草案必须包含：

- 适用范围；
- 前置条件；
- 按主题分组的检查项；
- 所需证据；
- 红旗问题；
- 关闭标准；
- 复核人签署或确认要求。

## 4. 输出格式

写入文件之前，先输出：

```text
拟创建文件：
- 路径：
  资产类型：
  创建理由：
  覆盖风险：
```

然后只创建用户批准的文件。

写入文件之后，输出：

```text
已创建文件：
- 路径：
  用途：
  下一复核负责人：

校验：
- 未包含保密个案事实
- 已包含人工复核点
- 已包含停止条件
```

## 5. 停止条件

出现以下情况时，必须在写入前停止并询问用户：

- 已选工作流包含未脱敏的保密事实；
- 目标路径已经存在；
- 资产类型不清楚；
- 缺少第一阶段审计输出；
- 候选工作流缺少评分或理由；
- 资产内容会固化具体法律意见，而不是沉淀工作流结构；
- 资产需要当前工作区之外的最新法律研究或法规检索。

## 6. 开始方式

请用户提供已选定的候选工作流和目标资产类型。若用户已经提供，则先列出拟创建文件，不要直接写入。
