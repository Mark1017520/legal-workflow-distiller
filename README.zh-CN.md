# 法律工作流蒸馏器

这是一个面向法律、合规、隐私、产品法务、监管事务和法律运营团队的 Codex / Claude Code / 本地工作区提示词项目。

它的目标不是让 AI 直接给法律意见，而是帮助法律人从近期完成过的工作中识别出可复用、可沉淀、可标准化的法律工作流，并判断这些工作流适合做成：

- Skill
- custom subagent
- automation
- template
- checklist
- 扩展已有资产
- 暂不沉淀

## 这是什么

本项目的核心是一个“只读审计”提示词：

```text
prompts/legal-workflow-distiller-v1.0.0.md
```

你把它复制到 Codex 或本地代码代理中运行，它会先扫描当前可访问的工作区材料，再输出候选法律工作流、评分、资产覆盖矩阵和 Top 3 资产化建议。

默认模式是：

```text
AUDIT_ONLY 只读审计模式
```

也就是说，在你明确回复“开始创建资产”之前，它不能创建、修改、删除、移动、重命名任何文件。

## 适合谁

- 公司法务
- 隐私合规团队
- 产品法务
- 合同审查团队
- 监管事务团队
- 法律运营团队
- 想把日常法律工作沉淀成 AI 工作流资产的人

## 它解决什么问题

很多法律工作不是一次性的，而是反复出现的：

- 合同初审
- DPA / PIA
- 数据出境判断
- 产品上线合规评审
- AI 功能审查
- 监管问询响应
- 开源许可证审查
- 营销文案审查
- 供应商安全和合同审查

这个项目帮助你回答：

- 哪些法律工作值得沉淀？
- 哪些工作适合做成 Skill？
- 哪些只适合做 template 或 checklist？
- 哪些工作风险太高，必须保留人工复核？
- 哪些已有资产可以复用或扩展？

## 怎么使用

1. 先阅读脱敏清单：

```text
docs/pre-run-sanitization-checklist.md
```

2. 复制核心提示词：

```text
prompts/legal-workflow-distiller-v1.0.0.md
```

3. 在要审计的工作区里打开 Codex。

4. 粘贴提示词并运行。

5. 等待输出：

- 可访问资料范围
- 现有资产覆盖矩阵
- 候选工作流评分表
- Top 3 资产化建议
- Skip 项
- 人工复核点
- 后续创建顺序

6. 如果确认要创建资产，再使用第二阶段提示词：

```text
prompts/legal-workflow-asset-builder-v1.0.0.md
```

## 输出长什么样

完整示例见：

```text
examples/sanitized-audit-output.md
```

它会像一份法律运营审计报告，而不是简单聊天总结。

## 哪些材料不能直接放进去

运行前请确认不要包含：

- 客户真实名称
- 员工姓名
- 个人邮箱、手机号、地址、身份证号
- 合同正文
- 争议事实
- 监管函件原文
- 未公开产品计划
- 真实 API key、token、cookie、日志密钥
- 受律师保密、调查保密或客户保密约束的材料

如果确实需要分析这类材料，请先确认你的 Codex 环境、公司政策和数据处理边界允许。

## 使用前要注意什么

- 这是法律工作流审计工具，不是法律意见生成器。
- 不要把输出直接当作最终法律结论。
- 高风险候选工作流必须由法律负责人复核。
- 可复用资产中只能沉淀流程、审查维度、输入要求、输出格式和人工复核点。
- 不要把个案事实、客户材料、合同原文或争议材料固化进通用资产。

## 推荐最小流程

1. 脱敏工作区材料。
2. 运行 audit-only prompt。
3. 查看候选工作流评分。
4. 只选择 Top 1-3 个高价值、低敏感、可复用工作流。
5. 用 asset-builder prompt 创建草案。
6. 人工复核后再放入团队知识库或 Codex Skills。
